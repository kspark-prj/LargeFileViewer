# Rust 가속 코어 빌드 가이드 (RUST_GUIDE.md)

이 프로젝트는 대용량 파일 인덱싱 및 정규식 검색 성능을 극대화하기 위해 Rust로 작성된 코어 모듈(`large_file_core`)을 지원합니다.

현재 파이썬 코드([LargeFileViewer.py]는 **자동 폴백(Fallback) 구조**로 구현되어 있어, Rust 코어가 빌드되지 않은 상태에서도 기존의 순수 파이썬 코드(`mmap`)로 문제없이 작동합니다.

Rust 가속을 활성화하려면 아래 단계에 따라 C++ 컴파일러(Linker)를 설치하고 빌드해 주세요.

---

## 🛠️ Rust 가속 코어 빌드 단계

### 1단계: Visual Studio C++ 빌드 도구 설치 (필수)

Windows 환경에서 Rust 코드를 컴파일하려면 C++ 링커(`link.exe`)가 필요합니다.

1. [Microsoft Visual Studio Build Tools 다운로드 페이지] 로 이동합니다.
2. 설치 관리자(Installer)를 다운로드하고 실행합니다.
3. 설치 화면에서 **"C++를 사용한 데스크톱 개발" (Desktop development with C++)** 워크로드를 선택합니다.
4. 우측 상세 정보 창에서 아래 컴포넌트가 선택되어 있는지 확인 후 설치를 진행합니다:
    - **MSVC v143 - VS 2022 C++ x64/x86 빌드 도구**
    - **Windows 11 SDK** (또는 Windows 10 SDK)
5. 설치 완료 후 컴퓨터를 재부팅합니다.

### 2단계: Rust 코어 빌드 및 설치

터미널(PowerShell 또는 Command Prompt)을 열고 프로젝트 폴더로 이동한 후, 다음 명령어를 실행합니다:

```powershell
# Cargo(Rust 패키지 매니저) 경로를 임시로 설정 (필요한 경우)
$env:PATH += ";$env:USERPROFILE\.cargo\bin"

# Rust 코어 패키지 빌드 및 Python 환경에 설치
pip install ./large_file_core
```

### 3단계: 가속 활성화 확인

빌드가 성공적으로 완료된 후 뷰어를 실행하면, 파일 로드 시 상단에 **`인덱싱 중... (Rust 가속 사용)`** 메시지가 표시되며 고속 처리가 활성화됩니다.

## ⚡ Rust 도입 시 성능 향상 원리

러스트 코어를 빌드하여 활성화했을 때 파이썬 대비 대용량 파일 처리 속도가 10배에서 최대 100배까지 빨라지는 물리적인 원리는 다음과 같습니다.

### 1. GIL(Global Interpreter Lock) 해제를 통한 "진짜 병렬 처리"

- **파이썬의 한계:** 파이썬은 GIL이라는 제약으로 인해 멀티스레드를 생성해도 실제로는 하나의 CPU 코어만 번갈아가며 사용합니다.
- **러스트의 강점:** GIL 제약 없이 `rayon` 라이브러리를 통해 대용량 파일을 여러 구역으로 나누어 **모든 CPU 코어가 동시에 검색을 수행(Parallel Search)**하므로, 코어 수에 비례하여 검색 속도가 기하급수적으로 빨라집니다.

### 2. SIMD(단일 명령 다중 데이터) 하드웨어 가속

- **파이썬의 한계:** 파이썬 루프를 통해 1바이트씩 줄바꿈 문자(`\n`)를 검사하며 리스트를 채우는 방식은 가상 머신(VM) 레벨의 오버헤드가 큽니다.
- **러스트의 강점:** `memchr` 크레이트를 사용해 CPU의 **SIMD(AVX2, AVX-512 등) 명령어**를 직접 호출합니다. CPU가 한 번의 명령 주기 동안 1바이트가 아닌 **32~64바이트를 통째로 검사**하므로 메모리 대역폭 한계에 준하는 속도로 개행 인덱싱을 완료합니다.

### 3. 극단적인 메모리 효율성 (CPU 캐시 친화 구조)

- **파이썬의 한계 (라인당 36바이트):** 파이썬의 정수형(`int`)은 28바이트 크기의 무거운 객체이며, 리스트는 이 객체들의 주소 포인터(8바이트)를 가리켜 정수 하나당 총 36바이트를 소모합니다. (1억 줄 기준 약 **3.6 GB** 메모리 낭비 및 CPU 캐시 미스 유발)
- **러스트의 강점 (라인당 8바이트):** 러스트의 `Vec<u64>`는 포장지 없이 순수 8바이트 정수가 메모리에 연속적으로 밀착 배치됩니다. 1억 줄 기준 단 **800 MB**만 소모(약 **78% 절감**)하며, 메모리가 연속적이어서 CPU 캐시 적중률이 극대화되어 데이터 접근이 극도로 빠릅니다.

### 4. Garbage Collector(GC) 없는 네이티브 기계어 실행

- **파이썬의 한계:** 실행 도중 메모리를 자동으로 정리하는 가비지 컬렉터(GC)가 주기적으로 동작하여 프로그램 연산을 미세하게 일시 중지시킵니다.
- **러스트의 강점:** 가상 머신(VM)이나 GC가 없으며, 컴파일 타임에 메모리 수명이 확정되어 런타임에는 기계어 수준에서 오버헤드가 제로(0)에 가깝게 동작합니다.

### 5. 바이트 단위 직접 정규식 매칭

- **파이썬의 한계:** 정규식 검색을 하려면 파일의 바이너리 데이터를 파이썬 문자열 객체로 변환(인코딩/디코딩)하는 비용이 듭니다.
- **러스트의 강점:** `regex` 엔진이 파일의 원본 바이너리 바이트 상에서 **디코딩 없이 즉각적으로 바이트 정규식 매칭을 수행**하므로 불필요한 문자열 객체 복사 및 데이터 변환 비용이 발생하지 않습니다.
